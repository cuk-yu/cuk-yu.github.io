时间复杂度

时间复杂度定义

时间复杂度是一个函数，定性描述该算法的运行时间

在软件开发中，时间复杂度作用：方便开发者估算出程序运行的答题时间。

如何估计程序的运行时间：

通常会估算算法的操作单元数量来代表程序消耗的时间，这里默认CPU的每个单元运行消耗的时间都是相同的

假设算法的问题规模为n,那么操作单元数量便用函数f(n)来表示,随着数据规模n的增大，算法执行时间的增长率和f(n)的增长率相同，称为算法的渐进时间复杂度，简称时间复杂度，记为O(f(n)).

什么是大O

大O是用来表示上界的

用它作为算法的最坏情况运行时间的上界，就是对任意数据输入的运行时间的上界

eg:

插入排序的时间复杂度：O(n²)
输入数据的形式对于程序运算时间是有很大影响的，在数据本来有序的情况下时间复杂度是O(n)，但如果数据是逆序的话，插入排序的时间复杂度就是O(n²)，也就是对于所有输入情况来说，最坏是O(n²)的时间复杂度，所以称插入排序的时间复杂度是O(n²)。
快速排序的时间复杂度：O(nlogn)
但是当数据已经有序的情况下，快速排序的时间复杂度是O(n²)，严格从O的定义上来讲，快速排序的时间复杂度应该是O（n²）

但是在业内称快速排序的时间复杂度是O(nlogn),这里的O代表的就是一般情况，而不是严格的上界。如图：

主要关心的还是一般情况下的数据形式

面试中说到算法的时间复杂度是多少值得都是一般情况

但是如果面试官和我们深入的探讨一个算法的实现以及性能的时候，就要时刻想着

数据用例的不一样

时间复杂度也是不同的，这一点一定要注意

不同数据规模的差异

如图可以看出不同算法的时间复杂度在不同数据输入规模下的差异

O(5n²) 和 O(100n) 两个时间复杂度，在数据规模为20之前，O(5n²)更优，也就是说随着数据规模的增大，开始最优的时间复杂度，不一定在数据规模增大后也是最优的时间复杂度

所以：
在决定使用哪些算法的时候，不是时间复杂度越低的越好（因为简化后的时间复杂度忽略了常数项等等），要考虑数据规模，如果数据规模很小甚至可以用O(n²)的算法比O(n)更合适（在有常数项的时候）。

为什么在计算时间复杂度的时候要忽略常数项系数呢，也就是说O(100n)就是O(n)的复杂度，O(5n²)就是O(n²)的复杂度，而且要默认O(n)优于O(n²)

这就涉及到大O定义了：

因为大O就是数据量级突破一个点且数据量级非常大的情况下所表现出的时间复杂度，这个数据量也就是常数项系数已经不起决定性作用的数据量

eg：上图中20的那个点，n只要大于20常数项系数已经不起决定性作用了

所以常说的时间复杂度都是省略常数项系数的，是因为一般情况下都是默认数据规模足够的大，基于这样的事实，给出的算法时间复杂度的排序如下：

O(1)常数阶 < O(logn)对数阶 < O(n)线性阶 < O(n²)平方阶 < O(n³)立方阶 < O(2^n)指数阶

但是也要注意大常数，例如10^7,10^9,那么常数就是不得不考虑的因素了

复杂表达式的化简

如何描述算法的时间复杂度是一个复杂表达时算法的时间复杂度
eg:
O(2n^2 + 10n + 1000)
简化法：
1.去掉运行时间中的加法常数项(因为常数项并不会因为n的增大而增加计算机的操做次数)
O(2n^2 + 10n)
2.去掉常数系数
O(n^2 + n)
3.只保留最高项，去掉数量级小一级的n（因为n²的数据规模远大于n）
4.最终简化为：
O(n^2)

O(logn)中log是以什么为底？

底可以不同，为10、2、20等

此处说下忽略底数的描述

以2为底n的对数 = 以2为底10的对数 * 以10为底n的对数

忽略常数项在此处即可描述为忽略底数
